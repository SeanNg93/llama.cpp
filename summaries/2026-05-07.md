# llama.cpp daily summary — 2026-05-07

A busy day dominated by multimodal/audio model additions and converter infrastructure improvements. Twelve commits landed, with notable new model support for MiniCPM-V 4.6 and IBM Granite Speech, a CPU kernel fusion optimization, and several converter/build tooling cleanups.

## Multimodal / Audio

- **MiniCPM-V 4.6 support** — adds full vision encoder pipeline for MiniCPM-V 4.6 including flash-attention-compatible graph and converter support (#22529, `2496f9c`)
- **Granite Speech (granite-4.0-1b-speech) support** — adds IBM Granite audio encoder (Conformer + QFormer projector, 80-bin mel spectrogram, 2× frame stacking); token-for-token verified against HF reference (#22101, `a00e47e`)

## CPU Performance

- **Fuse RMS_NORM + MUL on CPU backend** — kernel fusion reduces memory bandwidth pressure for common transformer patterns; potential throughput improvement for CPU inference (#22423, `f08f20a`)

## OpenCL / GPU Fit

- **OpenCL: use `CL_DEVICE_GLOBAL_MEM_SIZE` for `--fit` memory estimate** — fixes model layer allocation when using OpenCL devices (#22688, `2ca1161`)
- **Common: skip `--fit` for unknown device memory** — prevents incorrect layer assignments when device memory is not queryable (#22614, `a010122`)

## Convert / GGUF

- **gguf-py: bump to v0.19.0** (#22664, `a290ce6`)
