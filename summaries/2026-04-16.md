# llama.cpp daily summary — 2026-04-16

Today saw 11 commits merged to upstream/master, with heavy activity across GPU backends (CUDA, Vulkan, WebGPU), a significant new RPC transport, and several multi-GPU infrastructure improvements. Two commits stand out as potentially impactful: the new Q1_0 CUDA backend and the addition of native RDMA/RoCEv2 transport for the RPC backend.

## CUDA/GPU

- **New: Q1_0 CUDA backend** — initial CUDA kernels for Q1_0 quantization, including MMQ and MMVQ paths with dp4a support; AMD MMA guard included. Notable performance on RTX 5090 (12k+ t/s pp512 for 8B models). (#21629, `7e72b38`)
- **CUDA: NCCL communicator management refactored** — moved into context, lazy init limited to >1 GPU, fixed AllReduce hang when only a subset of devices are used; supersedes #21746. (#21891, `014dca4`)
- **CUDA: explicit opt-in for P2P access** — P2P GPU memory access now requires explicit opt-in; build docs updated. Potential behavioral change for multi-GPU setups. (#21910, `a620695`)

## Vulkan

- **im2col optimization** — improved memory write layout in the im2col shader, vendor-id-based tuning, capped workgroups. (#21713, `b3d7587`)

## WebGPU

- **Fix dequantization helpers** — refactored WGSL shaders to avoid pointer parameters (broke Firefox due to missing `unrestricted_pointer_parameters` extension); also increases XIELU precision. (#21872, `20d3bc2`)

## RPC

- **Native RDMA transport (RoCEv2)** — adds a new RDMA-based transport layer for the RPC backend, enabling high-throughput low-latency GPU-to-GPU communication over RoCEv2 networks. Large addition (~600 lines). (#20590, `adb541a`)

## API/server

- **Random media marker** — server now generates a random 32-char alphanumeric marker per launch instead of the fixed `<__image__>` token, preventing collisions with model output. (#21962, `408225b`)

## Parsing / autoparser

- **JSON_NATIVE + per-call markers** — autoparser now handles the `JSON_NATIVE` case with per-call markers; adds Reka-Edge template and tests. (#21892, `e1a9a6d`)

## Fixes

- **diffusion CLI: read n_ctx after context creation** — fixes crash where `n_ctx=0` caused all inputs to be rejected without explicit `-c`. (#21939, `e39eba2`)

## Docs

- **RoPE documentation** — significantly expanded `ggml_rope_ext` API docs and HOWTO-add-model guide to reduce misuse. (#21953, `80d8770`)

## CI

- Disabled `test-backend-ops` on Vulkan llvmpipe CI run and restored default timeout. (#21901, `8dc530b`)
