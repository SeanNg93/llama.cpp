# llama.cpp daily summary — 2026-04-26

A moderately active day with 9 commits across multiple GPU backends (CUDA, Metal, WebGPU, SYCL, Hexagon), a notable default-quantization change, and several bug fixes in chat and speculative decoding. One commit introduces a **potentially breaking change** to the default quantization type for programs relying on library defaults.

## CUDA/GPU

- **CUDA MMQ stream-k overhead reduction** — refactors `mmq.cuh` to use 32-bit integers for `kbc` and restructures warp scheduling, reducing launch overhead for stream-k MMQ kernels. (#22298, `9725a31`)

## Metal

- **Metal Tensor API optimization for `GGML_OP_MUL_MAT`** — splits the Metal Tensor API (`matmul2d`) path into its own standalone kernel gated by `GGML_METAL_HAS_TENSOR`, eliminating legacy layout constraints that previously prevented memory and dispatch optimizations. Large diff (~300 lines changed). (#20962, `d164904`)

## WebGPU

- **SSM_SCAN support** — implements the `SSM_SCAN` op in the WebGPU backend via a new WGSL shader; also disables `set_rows` error checking and removes blocking in graph compute. Ops support table updated. (#22327, `dd2914d`)

## SYCL

- **Q4_0 `mul_mat` optimized for Intel Arc 770** — tunes block/thread parameters for Arc770 hardware and adds helper scripts (`start-svr.sh`, `win-start-svr.bat`, `win-test.bat`) for running the server and benchmarks on Linux and Windows. (#22291, `eddd7a1`)

## Hexagon

- **HMX clock guard for v75+ platforms** — adds a compile-time guard so the HMX clock request is only issued on Hexagon v75 and later, preventing failures on older targets. (#22377, `b760272`)

## Chat / Inference

- **Fix space handling in reasoning markers** — corrects off-by-one / whitespace stripping logic in `chat-diff-analyzer.cpp` for models that include a space before/after reasoning delimiters. (#22353, `dcad77c`)
- **Speculative decoding vocab compat fix** — tightens vocabulary compatibility checks in `speculative.cpp` to avoid silent mismatches when draft and target models have different vocab sizes. (#22358, `98dc141`)

## Quantization ⚠️

- **Default `ftype` changed from `Q5_1` → `Q8_0`** — `llama_model_quantize_params` now defaults to `LLAMA_FTYPE_MOSTLY_Q8_0`. Programs that call `llama_model_quantize` without explicitly setting `ftype` will now produce Q8_0 output instead of Q5_1. **Potential breaking change for downstream tooling.** (#20828, `9d34231`)

## Misc / Build

- **gitignore + PR template** — adds `.pi` directory and personal `SYSTEM.md` to `.gitignore`; minor wording fix in the PR requirements heading. (#22316, `8ea8fee`)
